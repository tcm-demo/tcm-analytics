产品需求文档：商品列表管理

Product Requirements Document — Goods List Management
📅 版本 v1.4 📌 状态 迭代中 👤 产品：产品通 📆 日期 2026-06-30

1 问题陈述

用户痛点

生鲜零售门店运营中，店长和收银员每天需要频繁与商品数据打交道。当前系统缺少一个集中化的商品管理入口，导致以下问题：

  • 商品分散，缺少全局视图：商品散落在不同门店和分类下，店长无法从一个页面总览全部在售商品，不清楚哪些商品缺货、哪些已下架。
  • 筛选维度不足：仅有简单的搜索框，无法按「门店」「上下架状态」「启用/禁用状态」「售卖类型」「商品分类」等维度组合筛选，查找特定商品效率低。
  • 新增/编辑流程割裂：新增商品需要在多个页面跳转（先录入基本信息，再去另一页配图/调价），操作链路长且容易遗漏。
  • 批量操作缺失：季末或促销结束后需要批量下架商品时，只能逐个操作，耗时且容易出错。
  • 与商品分类体系未打通：分类管理页面维护了分类后，商品列表无法直接联动筛选，需要手动搜索分类名。

不解决的代价

  • 运营效率低下：店长每日商品维护耗时 30 分钟以上，纯手工操作。
  • 数据一致性风险：不同门店独立管理商品，导致同一 SKU 在不同门店的规格、价格不一致。
  • 分销/促销响应慢：批量上下架无法快速执行，影响季节性商品的上新和下架节奏。

2 目标 (Goals)

#目标类型衡量方式
G1 店长可在 10 秒内通过多维度筛选定位到目标商品 操作效率 从打开页面到找到目标商品的平均耗时
G2 新增一个商品（含规格、价格、分类）的填表时间不超过 2 分钟 用户体验 新增弹窗的填表耗时 + 必填校验通过率
G3 批量上下架 N 件商品的操作可在 5 秒内完成 操作效率 勾选 → 点击「下架」的交互次数 ≤ 3 次
G4 商品分类侧边栏与分类管理页面保持同步，筛选联动即时生效 功能完整性 分类增删改后，商品列表侧边栏自动更新无需刷新
G5 门店人员可见本店商品，企业管理员可查看全局数据 权限正确性 不同角色登录后看到的数据范围符合权限模型

3 非目标 (Non-Goals)

#不做的事项原因
N1 商品信息的批量导入/导出（Excel/CSV） 当前商品量级（每家门店数十到数百 SKU）手工维护足够；导入导出需额外处理数据清洗和校验，ROI 不高
N2 商品历史价格曲线和改价日志 已有独立的「改价日志」页面，不在商品列表页面重复实现
N3 商品库存管理 库存属于独立的供应链模块，涉及采购、入库、盘点等流程，与商品列表的"基础信息管理"定位不同
N4 标签体系的完整 CRUD 当前标签在商品编辑时从预设列表选择；标签增删改的管理功能为独立模块，不在本页实现
N5 会员价的规则引擎（如「会员价 = 售价 × 95%」自动计算） 会员价策略涉及营销规则引擎，复杂度高，当前仅作展示

4 用户故事 (User Stories)

4.1 企业管理员

US1 — 作为企业管理员
我希望在一个统一页面查看所有门店的所有商品（表格形式，含商品名称、编码、规格、价格、分类、状态），
以便掌握全公司商品概览，及时发现异常（如某商品多店均下架）并调整策略。
US2 — 作为企业管理员
我希望通过左侧分类侧边栏按分类快速筛选商品，侧边栏同步展示每个分类下的商品数量，
以便快速了解各大类商品的 SKU 分布，并一键查看某个分类下的全部商品。
US3 — 作为企业管理员
我希望通过门店、售卖类型（售卖品/非售品）、上下架状态、启用/禁用状态、关键词等多个维度组合筛选商品，
以便在促销结束后快速定位需要下架的特定商品，或在换季时统一调整季节性商品的上下架状态。
US4 — 作为企业管理员
我希望在商品列表中勾选多条商品后，一键执行批量「上架」「下架」「启用」「禁用」操作，
以便在季节性调货或促销结束时批量处理商品，避免逐条操作的重复劳动。
US5 — 作为企业管理员
我希望通过弹窗新增商品时，能从系统商品库搜索选择已有商品（带出商品名和规格），也能新增标品（含 69 码条码和品牌），
以便快速录入新商品，复用系统已有的商品数据减少重复录入。
US6 — 作为企业管理员
我希望点击表格中的某行商品时弹出编辑弹窗，可以修改商品的分类、价格、门店、上下架状态等信息，
以便快速纠错或调整单个商品的属性，无需切换到其他页面。

4.2 门店人员

US7 — 作为门店收银员/店长
我希望打开商品列表时默认看到本门店的商品，并且能按分类快速筛选，
以便在结账时快速查找顾客询问的商品信息（如价格、规格）。
US8 — 作为门店店长
我希望能在商品列表中调整本店商品的上下架状态和售卖类型，
以便根据实际库存情况和销售策略灵活控制商品的可售状态。

4.3 边界用例

⚠️ 边界用例 — 无数据状态
当筛选条件匹配不到任何商品时，表格区域显示「暂无数据」提示而非空白页面，
以便我知道筛选条件没有匹配结果，而非系统故障。
⚠️ 边界用例 — 分页边界
当商品总数不足一页时，分页组件不显示页码（仅显示总数）；当超过一页时提供上下页和页码跳转，
以便在大量商品时高效翻页，少量商品时界面不冗余。
⚠️ 边界用例 — 批量操作无选中
当我点击「上架」「下架」等批量按钮但没有勾选任何商品时，系统提示「请至少选择一条商品」，
以避免我误以为操作已执行。
⚠️ 边界用例 — 删除分类后的商品归属
当企业管理员在分类管理页面删除一个分类后，该分类下的商品自动归入「未分类」而非消失，
以确保商品数据不丢失，且店长在商品列表中仍能找到这些商品。

5 需求明细 (Requirements)

5.1 页面布局

商品列表页面采用左右分栏布局：

  • 左侧分类侧边栏（固定宽度 200px）：展示全部分类树，每项显示分类名 + 颜色圆点 + 商品数量徽标。点击某个分类触发商品列表筛选；点击「全部」取消分类筛选。侧边栏顶部标题「商品分类」。
  • 右侧主内容区（弹性宽度）：包含筛选栏、操作按钮栏、数据表格、分页栏。

5.2 筛选栏

筛选项控件类型默认值说明
门店 下拉选择 (select) 全部门店 门店列表动态获取；默认显示全部，企业管理员可切换查看特定门店商品
售卖类型 单选按钮组 (radio group) 全部 可选值：全部 / 售卖品 / 非售品
上下架状态 分段按钮组 (radio group) 全部 可选值：全部 / 上架 / 下架。与售卖类型为 AND 关系
启用/禁用 分段按钮组 (radio group) 全部 可选值：全部 / 启用 / 禁用。控制商品的 status 字段（'0'=启用，'1'=禁用），与上下架（enable 字段）为独立维度，并列展示在上下架筛选后面
关键词搜索 输入框 (text input) 空 支持按商品名称、商品编码模糊匹配；按 Enter 触发搜索
操作按钮 重置 + 查询 — 「重置」恢复所有筛选条件为默认值；「查询」执行筛选并刷新表格

5.3 操作按钮栏

按钮类型优先级行为
新增商品 主要按钮 P0 打开「新增普通商品」弹窗（搜索系统商品 → 填写销售名称、分类、规格/单位、售价、原价、保质期、门店等）
新增标品 主要按钮 P0 打开「新增标品」弹窗（输入 69 码条码 + 标品名称(只读) + 销售品名(可编辑) + 分类 + 品牌 + 规格/单位 + 售价 + 原价 + 非售商品开关 + 门店选择）
上架 次要按钮 P0 批量将选中商品的 enable 状态设为 0（上架）。无选中时提示
下架 次要按钮 P0 批量将选中商品的 enable 状态设为 1（下架）。无选中时提示
启用 次要按钮 P0 批量将选中商品的 status 设为 0（启用）。无选中时提示
禁用 次要按钮 P0 批量将选中商品的 status 设为 1（禁用）。无选中时提示

5.4.2 表格交互

  • 行高亮：点击行高亮（浅蓝色背景），同一时间仅一行高亮。
  • 行点击编辑：点击高亮行弹出编辑弹窗，可直接修改该商品的属性。
  • 全选：表头 checkbox 勾选/取消全选当前页所有行（仅当前页，不跨页）。
  • 空状态：无数据时表格显示一行「暂无数据」占位，colspan=15。

5.5 分页栏

  • 位置：表格卡片底部，右对齐。
  • 元素：「共 N 条」统计 + 上一页 + 页码按钮（最多展示 7 个） + 下一页 + 「X 页」。
  • 默认每页：10 条。
  • 边界行为：首页时「上一页」disabled；末页时「下一页」disabled。
  • 翻页后：表格滚动到顶部，全选和高亮状态重置。

普通商品关键约束：
  • 普通商品不含：保质期、贮藏方式、非售商品开关（这些为标品专属字段）。
  • 系统商品必须从系统库选择（GL_SYSTEM_GOODS），保存时校验选中商品名是否在库中，防止自由输入。
  • 通过 PRODUCT_DEFS 字典维护每个系统商品的合理规格和默认分类，避免「猪肉配 mL」这类不合逻辑的组合。
  • 编辑态下系统商品设为 readOnly + 隐藏清除按钮，不可修改和删除；新增态正常可搜索。

5.7 新增/编辑弹窗（标品）

标品弹窗为 4 列网格布局（gl-form-grid-4），标签宽度统一 84px（gl-form-label-s）。条码字段编辑态 readonly 且清除按钮隐藏；标品名称始终只读。表单布局：

行列 1 (span2, 1/2)列 2 (span2, 1/2)列 3 (1/4)列 4 (1/4)
Row 1条码* — 文本输入框（编辑态 readonly，清除按钮隐藏）标品名称 — 只读文本输入框（readonly 灰底）
Row 2销售品名* — 纯文本输入框（不支持搜索下拉）商品分类 — 通用选择器品牌 — 文本输入框
Row 3规格/单位* — 规格输入 + 单位紧凑选择器保质期 — 输入框 + 天/月/年下拉贮藏 — 文本输入框
Row 4售价* — 数值输入框原价 — 数值输入框非售商品 — radio（是/否）门店 — 通用选择器
标品特有逻辑：
  • 非售商品：选「是」时，售价和原价设为 readOnly（保留用户输入值，灰底不可编辑），效果与标品名称一致。选「否」时售价为必填。
  • 销售品名：纯文本输入框，不支持搜索下拉。由条码查询后自动带入，或用户完全手动输入。保存时取销售品名值作为 salesName。
  • 条码：新增态可自由输入（支持扫码设备），编辑态 readonly + 清除按钮隐藏，不可修改。
  • 标品名称：始终只读 (readonly)，不可编辑，由条码查询或其他数据源自动填充。
  • 保质期格式化：保存时与规格合并为 spec/unit | 保质期+单位 格式存储到 goodsSpec 字段（如 "500g/袋 | 180天"）。
  • 规格兼容解析：打开编辑弹窗时从 goodsSpec 反向解析出规格、单位和保质期。

5.8 分类侧边栏联动

  • 数据源：侧边栏分类列表与分类管理页面共享同一数据源（GOODS_CATEGORIES），按 sort 值升序排列，过滤掉 hidden: true 的「未分类」。
  • 商品计数：每个分类旁显示该分类下的商品总数（实时计算，遍历 GL_MOCK_DATA 按 categoryId 匹配）。
  • 分类筛选：点击分类 → 设置 GL_FILTER_CATEGORY → 触发 glRenderTable() → 表格仅显示该分类商品；侧边栏高亮当前选中分类（浅蓝背景 + 蓝色文字）。
  • 「全部商品」：始终在侧边栏顶部，显示总商品数；点击后清空分类筛选恢复全部视图。
  • 外部跳转：分类管理页面点击分类卡片 → 调用 gcGoToGoodsList(catId) → 切换到商品列表页面 → 设置 GL_FILTER_CATEGORY = catId → 侧边栏自动高亮对应分类 → 表格筛选为该分类。
  • 筛选条件保留：GL_FILTER_* 为全局变量，页面切换时不重置；从分类管理跳转后自动设置分类筛选并刷新。
  • 颜色同步：侧边栏分类颜色圆点与分类管理页面一致，颜色与分类 ID 绑定（通过 getCategoryColor(catId) 获取）。
  • 隐藏分类不显示：标记为 hidden: true 的分类（如「未分类」）不在侧边栏出现。

5.9 角色权限控制

操作企业角色门店角色说明
查看商品列表✅ 所有门店✅ 仅本门店门店角色默认只能看到所属门店的商品
按分类筛选✅ 允许✅ 允许两角色均可使用分类侧边栏筛选
新增商品✅ 允许✅ 允许门店人员新增商品时门店自动设为本店（不可更改）
新增标品✅ 允许✅ 允许同上
编辑商品✅ 允许✅ 允许门店人员仅可编辑本门店商品
批量上下架✅ 允许⚠️ 本店门店人员仅可操作本门店商品
批量启用/禁用✅ 允许⚠️ 本店同上
实现要点：
  • 通过 CURRENT_USER_ROLE 判断角色（'enterprise' | 'store'）。
  • 门店角色登录时，商品列表自动筛选为本门店数据（侧边栏计数也仅统计本门店）。
  • 企业角色登录时，默认显示全部门店数据（近期交互改为倒序排列——最新上架的商品优先展示）。
  • 后续接入后端 API 时，写操作接口需在服务端再次校验权限。

行编辑路由规则

点击表格行时自动判断弹窗类型：

  • 若 goodsBrand 有值 或 goodsCode 以 69 开头 → 标品弹窗（glOpenStandardModal(data)）
  • 否则 → 普通商品弹窗（glOpenAddModal(data)）

筛选条件重置

「重置」按钮恢复所有筛选条件为默认值：分类=''、上下架=''、启用/禁用=''、门店=''、售卖类型=''、关键词=''、页码=1、高亮=null。同时恢复侧边栏高亮为「全部」并刷新分类计数。

8 API 接口定义

说明：所有接口统一使用 POST 协议，通过路径区分业务操作。本页面涉及的商品 CRUD、批量状态更新等接口需要在已有商品接口基础上进行调整后独立定义，本章仅列出本页面依赖的分类侧边栏相关接口。完整接口定义详见 商品分类管理 PRD 第 8 章。

8.1 接口概览

#路径权限说明
1 /api/v1/categories 企业 / 门店 查询分类列表（侧边栏数据源，按 sort 升序，含商品数量）
2 /api/v1/categories/{id} 企业 / 门店 查询单个分类详情（含商品数量）
3 /api/v1/categories/{id}/products/count 企业 / 门店 查询分类下商品数量（侧边栏徽标数据）
